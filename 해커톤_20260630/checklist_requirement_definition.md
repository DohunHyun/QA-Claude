# 요구사항정의서 체크리스트

**산출물명**: 요구사항정의서
**샘플 문서번호**: NHXXX-단위업무코드-AN06
**기준**: NH 표준 템플릿 (v1.0) — 실제 양식(`본문`/`작성가이드`/`작성예시` 시트) 반영 업데이트
**검증 단계**: 분석단계 (2차)

> ⚠️ **v1.1 변경 이력**
> - v1.0→v1.1: 실제 PoC 검증 사례(시정조치관리대장 CA_W26~29)에서 운영 중인 산출물이 시트명 `요구사항정의서(기능)` / `요구사항정의서(비기능)`로 **기능/비기능을 별도 시트로 분리 운영**하는 것이 확인됨. 단일 시트(본문/작성가이드/작성예시 구성) 표준 템플릿과 다른 실사용 변형이며, 현장에서는 분리형이 더 일반적으로 보임. 이에 따라 §4(기능 시트)와 §8(비기능 시트)을 "시트 존재가 전제된" 별도 검증 섹션으로 재정리하고, §8의 컬럼셋을 §4와 동일한 수준(Level1~3, 전제조건, 변경유형/변경이력 등)으로 확장함.
> - 검증 시 **표준 템플릿(시트명 `본문`)과 분리형 템플릿(시트명 `요구사항정의서(기능)`/`요구사항정의서(비기능)`) 두 가지 변형을 모두 인식**하도록 파서가 시트명 패턴을 유연하게 매칭해야 함.

### [개선] 변경 표기 규칙 준수

## 8. 비기능요구사항 시트 검증 (시트명: `비기능` 또는 `요구사항정의서(비기능)`)

### [개선] 비기능 시트 존재 여부
- 표준 시트(`비기능`) 또는 분리형 시트(`요구사항정의서(비기능)`) 중 하나가 반드시 존재하는가?

### [개선] 필수 컬럼 존재 여부 — §4와 동일 수준 적용
> 실제 결함사례(CA_W27~29)에서 비기능 시트가 기능 시트보다 컬럼이 단순화되어 운영되며, 그 결과 Level2/Level3·전제조건·변경유형·변경이력이 통째로 누락되는 사례가 반복 확인됨. 비기능 시트도 §4 기능 시트와 **동일한 필수 컬럼셋**을 적용한다.
- 요구사항ID / 요구사항명 / Level1 / Level2 / Level3 / 상세내용(또는 반영결과: 분석/설계/구현/테스트 단계별 문서명+페이지) / 전제조건 및 제약사항 / 변경내역-변경유형 / 변경내역-변경이력

### [개선] Level2/Level3 누락 일괄 점검
- 비기능 요구사항 전체 행에서 Level2 또는 Level3가 100% 공백인 경우(개별 누락이 아닌 컬럼 자체 미운용으로 의심) → 개선으로 별도 표기하여 "컬럼 미작성 패턴"임을 명시 (개별 행 단위 누락보다 우선순위 높게 보고)

### [개선] 변경유형/변경이력 누락 일괄 점검
- 비기능 요구사항 전체 행에서 변경유형 또는 변경이력이 100% 공백인 경우 동일하게 개선 (컬럼 미작성 패턴)

### [개선] ID 연속성
- 비기능 시트의 요구사항ID도 RQ-[단위업무코드]-[일련번호] 규칙을 따르며 기능 시트와 중복되지 않는가?

### [권고] 전제조건 공백 비율
- 전제조건 및 제약사항이 비기능 요구사항 전체에서 100% 공백인 경우 권고 (필수 항목은 아니나 비기능 요구사항 특성상 전제조건 기술 빈도가 높아야 함)

### [권고] 반영근거 구체성
- "아키텍처정의서 P11"처럼 문서명+페이지가 구체적으로 기재되었는가? (단순 "반영됨" 등 모호한 기술은 권고)
