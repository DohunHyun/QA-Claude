## 4. 정합성 검증 (프로그램 정합성)

### [개선] 배치목록(AN07) 포함 검증
- 배치목록(AN07)의 배치Job ID ⊆ 본 목록의 프로그램ID (배치 유형 프로그램으로 누락없이 등록되어 있는가)

### [개선] 인터페이스목록/정의서(AN08) 포함 검증
- 인터페이스정의서(AN08)의 본인시스템서비스ID(인터페이스 수행 프로그램) ⊆ 본 목록의 프로그램ID

### [개선] UI목록(DS01) 포함 검증
- UI목록(DS01)의 화면/보고서ID ⊆ 본 목록의 프로그램ID (화면/보고서가 프로그램으로도 등록되어 있는가)

### [개선] 프로그램목록 내부 중복
- 위 3개 목록을 통합하는 과정에서 동일 프로그램ID가 중복 등록되지 않았는가?

### [권고] 유형 식별 일관성
- 화면ID/인터페이스ID에 해당하는 프로그램이 기능유형 상에서도 올바르게 식별(O/S/B/M)되었는가?

### [권고] 개발구분 변경관리 적절성
- 신규/변경/유지/삭제 분류가 실제 변경 내역과 일치하며, 삭제 시 사유가 구체적인가?
