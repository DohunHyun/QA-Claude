# 브랜치 전략 (QA-GPT)

> 대상: 개발자 3명 (김홍규·김명호·현도훈), 모두 git 초보 + 모두 Claude Code 사용.
> 목표: **git을 몰라도** 서로 안 부딪히고, `main`은 항상 동작하게 유지.

---

## 핵심 원칙 (이 5개만 기억)

1. **`main`은 신성불가침** — 항상 동작하는 코드만. main에 직접 커밋/푸시 ❌
2. **작업은 항상 새 브랜치에서** — 시작 전 `main`을 최신화하고 브랜치를 판다.
3. **한 작업 = 한 브랜치, 짧게** — 하루 안에 병합하는 걸 목표로. 오래 끌수록 충돌↑
4. **끝나면 PR** → 팀원 1명이 확인 → `main`에 병합 → 브랜치 삭제.
5. **git이 막히면 그냥 Claude에게 시킨다** — 단, 되돌리기 어려운 명령(`reset --hard`, `push -f`)은 실행 전 꼭 확인.

---

## 브랜치 이름 규칙

```
<종류>/<이름>-<작업>          예) feat/hong-file-upload
<종류>/<이슈번호>-<작업>      예) feat/12-file-upload   ← 이슈와 자동 연결
```

- **종류**: `feat`(기능) · `fix`(버그) · `docs`(문서) · `chore`(설정·잡일)
- 이슈·PRD는 **GitHub Issues**로 관리하므로, 브랜치에 이슈번호를 붙이면 PR이 이슈에 자동 연결됨.

---

## 담당 영역 (충돌 최소화의 핵심)

spec 역할 분담대로 **폴더가 갈려서** 서로 다른 파일을 만지니 충돌이 거의 안 난다.

| 사람 | 영역 | 주로 만지는 곳 |
|------|------|----------------|
| 김홍규 | 백엔드 / API | `controller` · `service` · `repository` |
| 김명호 | AI 엔진 / 파일 파싱 | `infra/llm` · `infra/parser` |
| 현도훈 | 프론트 / 기획 | `frontend/` |

> 규칙: **내 영역 폴더만 만진다.** 남의 영역을 바꿔야 하면 먼저 말하고 조율.

---

## 매일 작업 흐름 (복붙용)

```bash
# 1) 새 작업 시작 — main 최신화 후 브랜치 생성
git switch main
git pull origin main
git switch -c feat/hong-작업명

# 2) ... Claude와 작업 ...

# 3) 저장하고 올리기
git add -A
git commit -m "feat: 무엇을 했는지 한 줄"
git push -u origin feat/hong-작업명

# 4) GitHub에서 PR 생성 → 리뷰 1명 → "Squash and merge" → 브랜치 삭제
```

## Claude에게 이렇게 시키세요 (git 명령 몰라도 됨)

- "main 최신화하고 `feat/내이름-작업명` 브랜치 새로 만들어줘"
- "지금까지 변경사항 커밋하고 푸시한 다음 PR 올려줘"
- "main의 최신 내용을 지금 브랜치에 반영해줘"
- "이 머지 충돌 해결해줘"
- "방금 한 커밋만 취소해줘 (파일 내용은 그대로 두고)"

## 초보자 5대 수칙

- ✅ 작업 전 **항상** `git pull`
- ✅ **작게 자주** 커밋·병합 (하루 넘기지 않기)
- ❌ `main`에 직접 커밋/푸시하지 않기
- ❌ 남의 브랜치에서 작업하지 않기
- ⚠️ `git reset --hard` · `git push -f`는 되돌리기 어려움 → 실행 전 반드시 확인

---

## ⚠️ 한 컴퓨터에서 Claude를 여러 개 돌릴 때 (중요)

**같은 폴더를 여러 터미널이 공유하면 위험하다.** git은 폴더 하나당 브랜치 하나만 체크아웃되므로, 한쪽에서 브랜치를 바꾸거나 `git add -A`로 커밋하면 **다른 쪽의 작업 중인 파일까지 휩쓸려 들어간다.**

해결책 (택1):

- **(A) git worktree — 권장.** 브랜치마다 폴더를 분리한다.
  ```bash
  git worktree add ../QA-Claude-featX feat/hong-featX
  # 이제 ../QA-Claude-featX 폴더에서 작업 → 원래 폴더와 안 겹침
  git worktree remove ../QA-Claude-featX   # 끝나면 정리
  ```
- **(B) 각자 저장소를 따로 `clone`** 해서 자기 폴더에서 작업. (3명이 각자 PC면 자연히 이 방식)
- Claude Code 자체 기능(서브에이전트 worktree 격리)으로도 분리 가능.

> 지금처럼 한 폴더에서 두 터미널이 동시에 작업 중이라면, **커밋할 때 `git add -A` 대신 파일을 콕 집어** (`git add 파일경로`) 넣어 서로의 미완성 작업이 섞이지 않게 한다.

## PR / 병합 규칙 (해커톤용 가볍게)

- PR 제목 = 브랜치가 한 일 한 줄 요약
- 리뷰어 1명이면 충분 (형식 확인 위주로 빠르게)
- 충돌 없으면 **Squash and merge** (커밋 히스토리 깔끔)
- 병합 후 브랜치 삭제 (GitHub 버튼 한 번)
