## 3. "기능" 시트 검증

### [개선] 필수 컬럼(매트릭스 헤더) 존재
- 요구사항ID / 요구사항명 / 단위업무명 / Level1~3 / 액티비티ID / 액티비티명 / UI ID / UI명 / 배치ID / 배치명 / 인터페이스ID / 인터페이스명 / 프로그램ID / 프로그램명 / **단위테스트 케이스ID/명 / 통합테스트 시나리오ID/명 / 인수테스트 시나리오ID/명**

> ⚠️ 실제 결함사례(CA_W30)에서 단위/통합/인수 테스트 매핑 컬럼이 시트에 존재하되 76개 요구사항 전체 100% 공백인 사례가 확인됨. 컬럼 "존재"와 "값 채움"을 분리하여 검증해야 함(아래 신규 항목 참조).

### [개선] 테스트단계 추적 컬럼 일괄 공백 점검
- 단위테스트/통합테스트/인수테스트 매핑 컬럼이 전체 요구사항 행에서 100% 공백인 경우 → "테스트 단계 양방향 추적성 미구축" 결함으로 별도 보고 (개별 행 누락보다 우선순위 높은 패턴결함)
- 분석/설계 단계 검증 시점에는 아직 테스트 산출물이 없을 수 있으므로 권고로 낮추되, **테스트 단계 검증 시점에는 개선으로 격상**

### [개선] 요구사항ID 형식 및 중복
- RQ(또는 RQM)-[업무코드(2)]-[일련번호(4)] 형식 준수, 중복 여부 확인

### [개선] 행 단위 추적 매핑 완전성
- 요구사항ID마다 최소 1개 이상의 액티비티ID가 매핑되어 있는가? (공백이면 추적 불가 → 개선)
- 분석 단계 종료 후에는 UI/배치/인터페이스/프로그램ID 중 관련된 항목이 매핑되어 있는가? (설계 단계 진행 후 미매핑 시 개선)

### [개선] 양방향 추적 — 정의서 측 결손
- 요구사항정의서(AN06)에 존재하는 모든 요구사항ID가 본 시트에 등장하는가? (미등장 시 개선 — §6.5 업무레벨 체크)

### [개선] 양방향 추적 — 추적표 측 결손
- 본 시트에 존재하는 모든 요구사항ID가 요구사항정의서(AN06)에 정의되어 있는가? (미정의 시 개선)

### [개선] 액티비티ID 정합성
- 본 시트의 액티비티ID가 프로세스정의서(AN04)의 액티비티ID와 일치하는가? (불일치/누락 시 개선)
- 액티비티ID가 서로 다른 요구사항에 동일하게 중복 부여되어 있지 않은가?

### [개선] 후행 산출물 ID 존재성 (셀 단위 교차검증)
- 매핑된 UI ID가 UI목록(DS01)에 실제 존재하는가?
- 매핑된 배치ID가 배치Job목록(AN07)에 실제 존재하는가?
- 매핑된 인터페이스ID가 인터페이스정의서(AN08)에 실제 존재하는가?
- 매핑된 프로그램ID가 프로그램목록(DS07)에 실제 존재하는가?
- (존재하지 않는 ID를 추적표에 기재한 경우 개선 — 매핑 오류)

### [권고] 멀티값 셀 표기 형식
- 한 셀에 여러 ID가 들어가는 경우(예: "UI-CO-0001 UI-CO-0002") 구분자가 일관되게 사용되었는가?

## 4. "비기능" 시트 검증

### [개선] 필수 컬럼 존재

### [개선] 비기능 요구사항ID 매핑 완전성

### [개선] 비기능 시트 단계별 매핑 컬럼 일괄 공백 점검
- 분석(프로세스)/설계(UI·배치·인터페이스)/개발(프로그램)/테스트(단위·통합·인수) 매핑 컬럼이 비기능 요구사항 전체에서 100% 공백인 경우 개선 (실제 결함사례 CA_W31: 매핑 컬럼 전체 누락)

### [권고] 반영결과 구체성

## 4-1. 개정이력 탭 세부 검증

### [개선] 승인자 컬럼 누락 점검
- 개정이력 모든 행에서 승인자가 100% 공백인 경우 개선 (실제 결함사례 CA_W32)

### [개선] 요구사항정의서와의 버전 동기화
- 본 문서 최신 버전·개정일자가 요구사항정의서(AN06)의 최신 버전·개정일자와 시점상 합리적으로 동기화되어 있는가? (실제 결함사례 CA_W33: 요구사항정의서 V1.0a/2026.01.13 vs 본 추적표 V0.1/2026.01.16 — 버전 표기체계 자체가 다르고 시점도 어긋나 추적표가 최신 변경을 반영했는지 확인 불가 → 개선)
- 버전 표기 체계(예: V1.0/V1.0a/V0.1 등)가 프로젝트 전체에서 통일되어 있는가? (불통일 시 별도 권고)
