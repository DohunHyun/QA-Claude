# 산출물 세트(인벤토리) 정합성 체크리스트

**적용 대상**: 단일 산출물 내부가 아니라, **목록형 산출물(N건)과 그에 대응하는 개별/세트형 산출물(M건) 간의 "건수·존재성" 비교**
**검증 단계**: 관리/분석/설계 전 단계 공통 — 개별 산출물 체크리스트(§본문 검증) 통과 후 **세트 단위 교차 검증**으로 별도 실행
**기준 사례**: 시정조치관리대장 실제 결함(CA_W06, CA_W17, CA_W23 등) — "목록에는 있는데 개별 산출물 파일/엔트리가 없다" 또는 그 반대 패턴

> 이 체크리스트는 개별 문서 체크리스트(예: `checklist_batch_job_list.md`)가 검증하는 "문서 내부 완전성"과는 다른 레이어입니다. 개별 문서가 각각 개선 없이 통과해도, **목록 건수와 실제 산출물 건수가 다르면** 별도로 이 체크리스트가 결함을 잡아내야 합니다. 실행 시점은 해당 목록·산출물 쌍이 모두 업로드된 직후(spec §7.2 Phase 3 "단순 목록/정합성 체크" 이후)입니다.

---

## 1. 비교 로직 공통 원칙

### [개선] 목록 ⊆ 실물 (목록에는 있으나 실물 누락)
- 목록형 산출물에 등재된 ID/건수가 N개일 때, 대응 산출물(또는 산출물 내 엔트리)이 N개 모두 존재하는가?
- 누락된 ID는 목록과 1:1로 식별하여 **"미제출"** 결함으로 보고

### [개선] 실물 ⊆ 목록 (실물은 있으나 목록 미등재)
- 실제 제출된 산출물(또는 엔트리)의 ID가 목록형 산출물에 모두 등재되어 있는가?
- 미등재 항목은 **"표준미준수"** 또는 **"내용오류/불명확"**(추적 근거 불명)로 보고

### [권고] 건수 합산 오차
- 목록 총 건수와 실물 총 건수가 다를 경우, 위 개선으로 분류되지 않는 "단순 합계 불일치"가 남아있지 않은가? (예: 동일 ID가 실물 측에 중복 존재해 건수만 우연히 맞는 경우)

---

## 2. 검증 대상 쌍 (현재 11개 체크리스트 범위 내)

### [개선] 배치Job목록(AN07) ⟷ 배치설계서(DS09)
- AN07에 등재된 배치Job ID 전체 건수 vs DS09(배치Job명세 시트 또는 개별 파일)에 실제 설계된 배치Job ID 건수
- 실제 결함사례(CA_W06): 배치Job목록 60건 중 37건에 대응하는 배치설계서가 통째로 부재 → 누락 ID 목록화하여 보고
- AN07의 개발구분="신규"/"변경"인 항목만 대상으로 우선 비교 (유지/삭제는 설계서 재작성 불필요할 수 있음 — 단, 이 경우도 "유지"이면 기존 설계서가 존재해야 함을 별도 확인)

### [개선] 인터페이스정의서(AN08) ⟷ 인터페이스설계서(DS10)
- AN08에 등재된 인터페이스ID 전체 건수 vs DS10에 실제 설계된 인터페이스ID 건수(1파일=1인터페이스 운영 시 파일 개수와 동일)
- 양방향 확인: AN08에 없는데 DS10에만 있는 ID (실제 결함사례 CA_W23 유사 패턴 — 정의서 없이 설계서만 존재)

### [개선] UI목록(DS01) ⟷ 프로그램목록(DS07)
- DS01의 화면/보고서ID 전체 건수 vs DS07에 프로그램으로 등재된 동일 ID 건수 (DS07 §정합성 검증의 건수 버전)

### [개선] 배치Job목록(AN07) ⟷ 프로그램목록(DS07)
- AN07의 배치Job ID 전체 건수 vs DS07에 등재된 동일 ID 건수

### [개선] 인터페이스정의서(AN08) ⟷ 프로그램목록(DS07)
- AN08의 본인시스템서비스ID(프로그램) 전체 건수 vs DS07 등재 건수

### [개선] 요구사항정의서(AN06) ⟷ 요구사항추적표(PM-310)
- AN06의 요구사항ID 전체 건수 vs PM-310 "기능"/"비기능" 시트에 매핑된 요구사항ID 건수 (이미 §개별 체크리스트의 "양방향 추적" 개선과 동일 로직이나, 본 항목은 **건수 차이를 별도 수치로 산출**하여 보고 — 예: "AN06 76건 중 PM-310 미매핑 5건")

### [개선] 테일러링결과서(PM-120) "적용" 산출물 ⟷ 실제 업로드 파일 목록
- 테일러링결과서 개발/관리 시트에서 테일러링여부="적용"으로 표기된 전체 산출물 건수 vs 실제 업로드된 파일 건수
- 미제출 산출물을 산출물명 단위로 목록화

## 3. 보고 형식

### [개선] 비교 결과 보고 항목
세트 비교 결과는 개별 결함과 별도로 다음 형식으로 보고한다:
- 비교쌍 (예: "배치Job목록 ⟷ 배치설계서")
- 목록 건수 / 실물 건수
- 누락 ID 목록 (목록에는 있으나 실물 없음)
- 초과 ID 목록 (실물에는 있으나 목록에 없음)
- 결함유형: **미제출**(누락) 또는 **표준미준수**(초과/근거불명)
