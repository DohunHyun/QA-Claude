# 프로세스정의서 체크리스트

**산출물명**: 프로세스정의서
**샘플 문서번호**: NHXXX-단위업무코드-AN04
**기준**: NH 표준 템플릿 (v1.0), 형식: PowerPoint (Level1~3 계층형 프로세스 트리)
**검증 단계**: 분석단계 (2차)

---

## 1. 파일명 검증

### [개선] 파일명 형식
- 규칙: NH[프로젝트코드]-[단위업무코드]-AN04-프로세스정의서_V[버전]_[작성일자(YYYYMMDD)].pptx

### [개선] 문서번호와 파일명 일치
- 표지 슬라이드의 문서번호(NHXXX-[단위업무코드]-AN04)가 파일명과 일치하는가?

---

## 2. 표지 / 개정이력 슬라이드 검증

### [개선] PM 등록 프로젝트정보 일치 검증
- 표지의 프로젝트명·문서번호 프로젝트코드가 PM 등록 정보와 일치하는가?
- 제·개정일자가 PM 등록 프로젝트 기간(착수일~종료일) 이내인가?

### [개선] 표지 필수 필드
- 프로젝트명 / 단위업무명("- 단위업무명 -" 치환 여부) / 문서번호 / 버전 / 제·개정일자

### [개선] 개정이력 필수 필드
- 버전 / 변경일 / 구분(최초작성/추가/수정/삭제) / 개정내용 / 작성자 / 승인자

---

## 3. 프로세스 트리(본문) 구조 검증

### [개선] Level 계층 구조 명확성
- 단위업무명 → Level1 → Level2 → Level3 계층이 명확히 구분되어 표현되는가?
- Level이 존재하지 않는 하위 단계는 NA로 명시되었는가? (공백 처리 시 개선)

### [개선] 시작/종료 정의 (Notion 표준)
- 각 프로세스 흐름도에 시작점과 종료점이 명확히 표기되어 있는가? (누락 시 개선 — 빈번 시정조치 사례)

### [개선] 표기 기호(Notion) 표준 준수
- 작업/분기/연결선 등 표기 기호가 사내 표준 Notion을 따르는가? (임의 기호 사용 시 개선)

### [개선] Input/Output 정보 명시
- 각 액티비티(프로세스 단위)에 입력값(Input)과 산출물(Output)이 명시되어 있는가? (누락 시 개선 — 빈번 시정조치 사례)

### [개선] 예외 처리 흐름 정의
- 정상 흐름 외 Error, 반려 등 예외 처리 흐름이 별도로 정의되어 있는가? (미정의 시 개선 — 빈번 시정조치 사례)

### [개선] 역할(담당부서) 명시
- 각 액티비티(스윔레인 등)에 수행 담당부서/역할이 명확히 표기되어 있는가? (불분명 시 개선)

---

## 4. 액티비티 ID 검증

### [개선] 액티비티ID 유일성
- 액티비티ID(예: AV-CO-LV-0001)가 서로 다른 프로세스에 중복 부여되지 않았는가? (빈번 시정조치 사례: 동일 액티비티ID/명이 중복 발견)

### [개선] 요구사항추적표와의 일치
- 본 문서의 모든 액티비티ID가 요구사항추적표(PM-310)의 액티비티ID 컬럼과 정확히 일치하는가? (불일치 시 개선)

### [권고] 액티비티명 명확성
- 액티비티명만으로 업무 내용을 이해할 수 있는가?

---

## 5. AS-IS/TO-BE 비교 (해당하는 경우)

### [권고] AS-IS 대비 TO-BE 변경 명확성
- 현행업무분석서/패키지GAP분석서 대체 자료로 사용되는 경우, AS-IS 대비 TO-BE 변경 내용이 명확히 기술되어 있는가? (테일러링결과서에서 "현행업무흐름도 → 프로세스정의서 TO-BE업무흐름도로 대체" 등 명시된 경우 필수 점검)

---

## 6. 추적성 검증

### [권고] 요구사항 대비 프로세스 커버리지
- 요구사항정의서의 모든 Level1/2/3이 본 문서의 프로세스 트리 상에 매핑되는가? (요구사항 카테고리 대비 프로세스 누락 여부)
