# 인터페이스설계서 체크리스트

**산출물명**: 인터페이스설계서(전문)
**샘플 문서번호**: NHXXX-단위업무코드-DS10
**기준**: NH 표준 템플릿 (v1.0), 형식: Excel
**검증 단계**: 설계단계 (3차) — 인터페이스ID 단위로 설계서가 작성됨 (1파일=1인터페이스 운영이 일반적)

> ⚠️ **v1.1 변경 이력**: 실제 PoC 검증 사례(시정조치관리대장 CA_W13~14)에서 운영 현장 표준 시트 구조가 **"표지/인터페이스정의/송신구조/수신구조" 4개 시트**임이 확인됨. 분석 당시 샘플(NHXXX 표준 템플릿)은 `본문(전문)` 단일 시트에 입력정의/출력정의를 모두 포함하는 구조였음. **두 변형이 실사용에서 혼재**하므로, 본 체크리스트는 두 구조를 모두 인식하여 검증하도록 §3을 재구성함.
> - **변형 A (단일시트형)**: 표지/개정이력/본문(전문) — 입력정의=수신측 관점, 출력정의=송신측 관점으로 한 시트에 통합
> - **변형 B (4시트형)**: 표지/인터페이스정의/송신구조/수신구조 — 연계방식·송수신 시스템 구분이 "인터페이스정의" 시트에 별도로 명시되고, 필드 정의가 송신구조/수신구조 시트로 분리

---

## 1. 파일명 검증

### [개선] 파일명 형식
- 규칙: NH[프로젝트코드]-[단위업무코드]-DS10-인터페이스설계서_전문_V[버전]_[작성일자(YYYYMMDD)].xlsx
- 1파일=1인터페이스 운영 시: 파일명에 인터페이스ID가 접두로 포함되는 경우(예: `IF-DV-0049_NBIA-DV-DS10-...`) 구분자는 **하이픈(-)** 통일을 원칙으로 하며 언더스코어(_) 사용은 개선 (실제 결함사례 CA_W03)

### [개선] 버전 표기 대소문자 통일
- 버전 표기가 프로젝트 전체에서 "V1.0" 대문자로 통일되어 있는가? (실제 결함사례 CA_W04: 인터페이스설계서 전량이 소문자 "v1.0" 사용, 타 산출물과 불일치)

### [개선] 동일 문서번호 중복 파일 점검
- 동일 인터페이스ID에 비표준 접미사가 붙은 중복 파일이 존재하는가? (실제 결함사례 CA_W05: `IF-DV-0030_..._v1.0_00.xlsx`처럼 `_00` 등 의미불명 접미사 파일 존재 → 개선)

### [개선] 문서번호와 파일명 일치

## 2. 표지 / 개정이력 검증

### [개선] PM 등록 프로젝트정보 일치 검증
- 표지의 프로젝트명·문서번호 프로젝트코드가 PM 등록 정보와 일치하는가?
- 제·개정일자가 PM 등록 프로젝트 기간(착수일~종료일) 이내인가?

### [개선] 표지 필수 필드 / 개정이력 필수 필드 및 일치

---

## 3. 시트 구조 검증 (필수 — 시트 변형 판별의 전제조건)

### [개선] 표준 시트 구조 존재 여부
- 변형 A(`본문(전문)` 포함 3시트) 또는 변형 B(`인터페이스정의`/`송신구조`/`수신구조` 포함 4시트) 중 하나의 구조를 온전히 갖추고 있는가?
- 단일 시트만 존재하며 위 두 구조 중 어느 것에도 해당하지 않는 경우 → **개선 "표준 시트 구조 누락"** (실제 결함사례 CA_W13: 58개 파일 전량이 단일 시트의 비표준 형식)

### [개선] 송수신 구분 정보 명시 (변형 B 기준 필수, 변형 A는 헤더에 준하는 정보로 대체 가능)
- 연계방식(실시간/배치), 송신시스템, 수신시스템 정보가 시트 내 명시되어 있는가? (누락 시 개선 — 실제 결함사례 CA_W14: 58개 전량 누락)

---

## 4. 인터페이스 헤더 검증 (변형 A: 본문(전문) / 변형 B: 인터페이스정의 시트)

### [개선] 헤더 필수 항목
- 단위업무명 / Level1 / Level2 / 인터페이스ID / 인터페이스명 / 수신 서비스 거래코드 / 결과수신 서비스 거래코드 / 에러처리 서비스 거래코드 / TIMEOUT 처리서비스 거래코드

### [개선] 인터페이스ID 형식 및 분석단계 정의서 존재
- 인터페이스ID가 IF-[업무코드(2)]-[일련번호(4)] 형식을 따르는가?
- 본 인터페이스ID가 인터페이스정의서(AN08)에 동일하게 존재하는가? (분석 단계에서 작성한 정의서를 기준으로 설계서를 작성하므로, 정의서에 없는 ID는 개선)

### [개선] 파일명-내부ID 일치
- 파일명에 표기된 인터페이스ID와 시트 내부(헤더 또는 시트명)에 기재된 인터페이스ID가 정확히 일치하는가? (실제 결함사례 CA_W15: 파일명은 IF-DV-0022이나 내부 시트명은 IF-FI-0056 — 파일 교체 누락으로 추정되는 불일치 → 개선)

### [개선] 거래코드 누락
- 수신/결과수신/에러처리 서비스 거래코드 중 해당 인터페이스 처리흐름 상 필요한 코드가 비어있는가? (TIMEOUT 처리는 온라인 동기식 인터페이스에서 필수)

---

## 5. 필드 정의 검증 (변형 A: 입력정의/출력정의 / 변형 B: 송신구조/수신구조 시트)

### [개선] 필수 컬럼 존재
- No / 필드명(한글명·영문명) / I-O 구분(변형 A) 또는 시트 구분(변형 B) / 필드속성 / 필드길이 / 소수점이하자리수 / 반복여부 / 반복횟수 / 설명

### [개선] I/O 구분 정확성 (변형 A)
- 입력정의 섹션의 모든 행이 I/O="I"로, 출력정의 섹션의 모든 행이 "O"로 표기되어 있는가? (혼재 시 개선)

### [개선] 필드 속성/길이 누락
- 모든 필드행에 필드속성(타입)과 필드길이가 채워져 있는가? (공백 시 개선 — 빈번 시정조치 사례: 데이터타입/길이 오류)

### [개선] 동일 한글명-이종 영문명 불일치 점검
- 동일한 필드 한글명에 서로 다른 영문명이 행마다 다르게 부여되어 있지 않은가? (실제 결함사례 CA_W16: 한글명 "임시내용1"에 영문명이 행별로 `IO_TMP_CNTN_1`, `TMP_CNTN1`로 혼재 → 개선)

### [개선] 반복여부-반복횟수 일치
- 반복여부="Y"인 필드에 반복횟수가 비어있으면 개선

### [개선] No 일련성
- No 컬럼이 1부터 누락/중복 없이 순차 부여되어 있는가?

### [권고] 필드명 누락 행 존재
- 표에 No만 있고 필드명·속성이 모두 공백인 빈 행이 다수 존재하는가? (템플릿 미작성 잔존 가능성)

---

## 6. 연계방식 정합성 검증 (인터페이스정의서 대비)

### [개선] 처리정보(데이터 항목) 일치
- 본 설계서의 입력/출력 필드 구성이 인터페이스정의서(AN08)의 처리정보 컬럼에 기술된 주요 데이터 항목과 상충되지 않는가?

### [개선] 연계방식 오류 (빈번 시정조치 사례)
- 인터페이스정의서 상 연계방법이 "온라인"(실시간)인데 본 설계서에 배치성 처리 로직(주기적 일괄처리)이 기술되어 있는 등 모순이 있는가? → 개선
- 송수신 시스템 간 거래코드 체계가 상호 일치하는가? (송신측 결과수신 거래코드 = 수신측 처리 거래코드)

### [권고] 오류/재처리 방안 명시
- TIMEOUT 또는 에러처리 거래코드가 정의된 경우, 재처리/재전송 로직이 설명(또는 별도 필드)에 기술되어 있는가?

---

## 7. 프로그램목록 대비 누락/오매핑 검증

### [개선] 프로그램목록 등재 대비 설계서 파일 존재
- 프로그램목록(DS07)에 인터페이스 관련 프로그램으로 등재된 인터페이스ID 중 본 설계서 파일이 존재하지 않는 ID가 있는가? (실제 결함사례 CA_W23 전반부: 프로그램목록에는 있으나 설계서 파일 없음 → 개선)

### [개선] 설계서 존재 대비 프로그램목록 등재 여부
- 본 설계서 파일이 존재하는 인터페이스ID가 프로그램목록(DS07)에 등재되어 있는가? (실제 결함사례 CA_W23 후반부: 설계서는 있으나 프로그램목록 미참조 → 개선)

### [개선] 프로그램목록 매핑 내용의 의미적 정합성
- 프로그램목록에서 특정 프로그램에 매핑된 인터페이스ID가 실제로 그 업무 내용과 일치하는가? (단순 ID 존재 여부가 아니라 **매핑 내용 검증** — 실제 결함사례 CA_W25: "통합로그인 사용자정보 생성"이 IF-DV-0046[실제는 인사정보]로 잘못 매핑된 사례, 정상은 IF-DV-0045)
