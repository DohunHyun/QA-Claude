# 배치Job목록 체크리스트

**산출물명**: 배치Job목록
**샘플 문서번호**: NHXXX-단위업무코드-AN07
**기준**: NH 표준 템플릿 (v1.0), 형식: Excel (시트: 표지/개정이력/본문/작성가이드)
**검증 단계**: 분석단계 (2차)

---

## 1. 파일명 검증

### [개선] 파일명 형식
- 규칙: NH[프로젝트코드]-[단위업무코드]-AN07-배치Job목록_V[버전]_[작성일자(YYYYMMDD)].xlsx

### [개선] 문서번호와 파일명 일치

---

## 2. 표지 / 개정이력 검증

### [개선] PM 등록 프로젝트정보 일치 검증
- 표지의 프로젝트명·문서번호 프로젝트코드가 PM 등록 정보와 일치하는가?
- 제·개정일자가 PM 등록 프로젝트 기간(착수일~종료일) 이내인가?

### [개선] 표지 필수 필드
- 프로젝트명 / 단위업무명 / 문서번호(NHXXX-[단위업무코드]-AN07) / 버전 / 제·개정일자

### [개선] 개정이력 필수 필드 및 표지 일치

---

### [개선] 배치Job ID 명명규칙
- 형식: BJ-[단위업무코드(2자리)]-[일련번호(4자리)], 0001부터 시작

### [개선] 배치Job ID 중복 확인
- 동일 shell 기준으로 여러 JOB이 등록된 경우(=중복) 1개만 "유지", 나머지는 개발구분="중복"으로 분류되어 있는가? (분류 누락 시 개선)

### [개선] 필수 필드 채우기
- 단위업무명/Level1/배치JobID/업무명/RUNID/작업종류/수행서버/경로/파일명/수행시작시각/수행주기/개발구분/농협IT담당자/수행사담당자

### [개선] 선후행 작업조건 상호 일치
- A의 후행작업조건에 B(RUN ID)가 기재되어 있으면, B의 선행작업조건에도 A가 기재되어 있는가? (불일치 시 개선)

### [개선] 개발구분별 조건부 필수
- 개발구분="삭제" 또는 "중복" → 비고에 사유 필수
- 개발구분="유지" 또는 "변경" → 현행RUNID 필수

## 4. 운영 안정성 검증 (빈번 시정조치 사례)

### [개선] 예상 소요시간 및 임계시간 정의
- 운영 안정성이 중요한(선후행 의존성이 있는) JOB에 예상수행시간과 임계시작/종료시각이 정의되어 있는가? (미정의 시 개선 — 설계단계 점검 시 더 중점적으로 확인)

### [개선] 실패시 처리방안 정의
- 온라인개시전필수완료여부="Y"인 JOB에 실패시처리방안이 비어있으면 개선 (업무 영향도가 큼)

### [권고] 반복작업 요건 명확성
- 반복작업요건이 있는 경우 시작시간/한계시간/반복횟수가 모두 구체적으로 기재되었는가?

---

## 5. 정합성 검증 (배치설계서 대비)

### [개선] 배치설계서(DS09)와의 ID 정합성
- 본 목록의 배치Job ID ⊆ 배치설계서(DS09)의 배치Job ID (설계 단계 이후 검증 시 적용, 분석단계 시점은 보류 가능)

### [권고] 요구사항 대비 배치 업무 누락
- 요구사항정의서 상 배치성 업무로 식별된 항목이 본 목록에 모두 반영되어 있는가?
