### [개선] 화면/보고서ID 중복 확인
- 동일 ID가 2개 이상 존재하면 개선

### [개선] enum값 준수
- 프로그램유형: "화면"/"보고서" 외 값 → 개선
- 기능유형: 프로그램유형="화면"일 때 메인화면/인클루드화면/팝업 외 값, ="보고서"일 때 전표/장표/출력물/기타 외 값 → 개선

- 리스크관리영역: ITA/STTA/STA/FTA 외 값 → 개선
- 개발구분: 유지/변경/삭제/신규 외 값 → 개선

### [개선] 보고서-관련화면ID 필수
- 프로그램유형="보고서"인 경우 관련프로그램/화면ID가 공백이면 개선 (다수인 경우 쉼표 구분)

### [개선] 삭제 사유 필수
- 개발구분="삭제"인데 비고가 공백이면 개선

## 4. 정합성 검증 (교차 산출물)

### [개선] 요구사항추적표(PM-310)와의 UI ID 정합성
- 요구사항추적표에 매핑된 모든 UI ID가 본 목록에 존재하는가? (누락 시 개선)
- 본 목록의 UI ID가 요구사항추적표에 매핑되어 있는가? (역방향 — 출처 불명 화면 식별)

### [개선] 프로그램목록(DS07) 포함 여부
- 본 목록의 모든 화면/보고서ID가 프로그램목록의 프로그램ID 목록에 포함되어 있는가? (누락 시 개선)

### [권고] AS-IS 대비 신규/삭제 적절성
- 분석단계 말 시점 TO-BE 유지 대상 기준으로 작성되었으며, 설계 이후 삭제분이 비고에 변경이력으로 관리되는가?

### [권고] 중요도 분포 점검
- 권고 비율(상20%/중50%/하30%) 대비 심각하게 편중되어 있는가?
