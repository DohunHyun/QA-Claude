## 📊 슬라이드별 발표 대사

### 🔷 슬라이드 1: 표지 (40초)

```
현재 SI 프로젝트의 품질검증(QA) 프로세스는 
평균 6~7일이 소요되는 것이 표준입니다.

당사가 개발한 QA-GPT는 산출물 1개 기준 동일한 검증을 5분 이내에 완료합니다.

본 솔루션은 AI 자동화, 규칙 기반 검증, 전문가 최종 검수를 결합하여 검증의 신뢰성과 효율성을 동시에 달성합니다.

오늘 그 구체적인 내용을 설명드리겠습니다.
```

### 🔷 슬라이드 2: 목차 (30초)

```
본 발표는 9가지 항목으로 구성됩니다.

현행 QA 프로세스의 문제점을 분석하고,
당사 솔루션의 3V 혁신 전략(Volume, Velocity, Variety)을 설명합니다.

기술 아키텍처, 검증 기준, 동작 절차를 순서대로 소개한 후
솔루션의 차별성, ROI 분석, 사업화 전략을 제시합니다.
```

### 🔷 슬라이드 3: 현행 문제점 (50초)

```
현재 SI 프로젝트 QA 프로세스는 4가지 구조적 한계를 보유합니다.

첫째, 시간 지연입니다.
당사 QA와 협업을 진행할 경우 6~7일의 검증 기간이 필요하며 이는 프로젝트 일정을 압박하고,
결함 발견이 지연되면서 추가 비용이 발생합니다.

둘째, 수작업의 물리적 한계입니다.
다향한 종류 및 수만 줄 규모의 산출물을 전수 검사하기는 현실적으로 불가능합니다.

셋째, 검증 품질의 편차입니다.
이부분이 중점적으로 고민된 부분으로 PM이나 QA담당자의 경험과 역량에 따라 동일 산출물의 검증 결과가 달라지는 문제가 있습니다.

넷째, 후속 비용의 폭증입니다.
당연하게도 설계 단계의 결함을 놓치면 개발, 테스트 단계에서 수정 비용이 기하급수적으로 증가합니다.
```

### 🔷 슬라이드 4: 3V 혁신 전략 (60초)

```
당사 솔루션은 3가지 혁신 축을 기반으로 개발되었습니다.

첫 번째는 VOLUME(규모)입니다.
11개의 평가 산출물에 NH 표준에 근거한 212개의 검증 규칙을 데이터베이스로 구축했습니다.

이를 통해 대상 산출물에 대한 일관성 있고 포괄적인 검증이 가능합니다.

두 번째는 VELOCITY(속도)입니다.
AI 에이전트의 병렬 처리와 자동화된 규칙 적용을 통해모든 검증 프로세스를 5분 이내에 완료합니다.

세 번째는 VARIETY(다양성)입니다.
AI 자동 검증, 규칙 기반 검증, 최종적으로 전문가 검수를 통하여 3단계 다층 검증을 실시합니다.
이를 통해 오류 최소화와 신뢰성 보증을 동시에 달성합니다.

3V 전략이 금번 솔루션의 핵심입니다.
```

### 🔷 슬라이드 5: 기술 아키텍처 (50초)

```
기술 아키텍처는 다음과 같이 설계되었습니다.

첫 번째 단계는 AI 자동 검증입니다.
표준 규격 검증, 요구사항 추적, 문서 간 일관성, 
산출물 통합 패키징의 4가지 특화된 에이전트가
동시에 병렬 처리합니다.

두 번째 단계는 규칙 기반 검증입니다.
212개 NH 표준 규칙을 결정론적으로 적용하여
매회 동일한 검증 결과를 보증합니다.

세 번째 단계는 전문가 최종 검수입니다.
AI 생성 결과를 전문가가 검토하여
최종 승인을 담당합니다.

이 구조를 통해 자동화와 신뢰성의 균형을 달성합니다.
```

### 🔷 슬라이드 6: 검증 기준 (40초)

```
검증 기준 구성을 설명드리겠습니다.

11종 산출물을 관리(3종), 분석(4종), 설계(4종) 3개 범주로 분류했습니다.

212개 검증 항목 중 185개, 즉 87%가 자동화된 정밀 패턴 매칭 엔진으로 처리됩니다.

자동화율을 보면 요구사항 추적표 95%, 요구사항 정의서 85%, 인터페이스 정의서 90% 수준입니다.

나머지 13%는 도메인 전문가의 수동 검증을 통해 최종 정확도를 보증합니다.
```

### 🔷 슬라이드 7: 동작 절차 (50초)

```
실제 동작 절차는 다음과 같습니다.

첫 단계에서 PM이 프로젝트 정보와 기준 표준을 등록합니다.
두 번째, 산출물 파일을 업로드합니다.
세 번째, AI 검증과 보고서 생성이 5분 이내에 완료됩니다.
네 번째, 단계별 최종 결과물을 수령합니다.

결과물은 4가지 형태입니다.
첫째, 시정조치 대장: 지적·조치·이력 자동 연동 추적 엑셀입니다.

둘째, AI 개선 산출물: 규격에 맞게 자동 보정된 교체용 파일입니다.

셋째, 비교표: 원본 vs AI 개선 산출물 변경점 대조입니다.

넷째, 단계 종합결과서: 발주사·감리 대응용 법적 효력급 공식 문서입니다.
```

### 🔷 슬라이드 8: 솔루션 차별성 (45초)

```
솔루션의 차별성을 비교 분석하겠습니다.

기존 수작업 방식은 샘플링 검증만 가능하고 개인 경험에 의존하며,
담당자별 편차가 발생하고 수작업으로 산출물을 작성합니다.

범용 LLM은 도메인 가이드가 없고 
단일 패턴만 적용하며, 매회 불일치한 결과를 산출합니다.

당사 QA-GPT는 다음의 우위를 갖습니다.

212개 규칙 RAG를 통한 도메인 특화 검증,
다차원 전수 검증으로 100% 적용률을 실현,
동일 입력에 대해 100% 동일한 결과 보증,
자동으로 3+1종 산출물 즉시 생성,
그리고 무엇보다 5분 이내 완료가 목표입니다.
```

### 🔷 슬라이드 9: ROI 분석 (55초)

```
경제 효과를 정량적으로 분석했습니다.

NH의 연간 QA 소요는 40회입니다.
20개 프로젝트 × 2회 검토 기준입니다.

현재 평균 소요 시간은 5.5일이고,
1일 인력비용 기준은 약 700만원입니다.

계산하면 연간 절감액은 1억 5,400만원입니다.

이는 보수적 추정입니다.
재작업 비용 절감, 지체상금 회피 등 추가 효과는 미반영된 수치입니다.

정성적 효과도 주목할 가치가 있습니다.

일관된 검증 기준으로 팀 역량의 표준화를 달성하고,
고객사에는 신속하고 완벽한 품질 보증으로 신뢰도를 강화할 수 있습니다.

이를 무한의 신뢰자산이라 평가합니다.
```

### 🔷 슬라이드 10: 확장 전략 (60초)

```
사업화 전략을 3단계로 구성했습니다.

Phase 1: 사내 표준화 플랫폼입니다.
당사 PaaS 환경에서 구축하여 모든 공공·금융 IT SI 프로젝트에 필수 탑재합니다.

효과는 QA 리소스 투입비의 원천 제거,
PM 역량 격차의 상향 표준화,
전 프로젝트에 동일 품질 기준 적용입니다.

Phase 2: 수주 제안 차별화 우위입니다.
RFP 발표 시 AI 기반 자동 품질 검증 능력을 차별화 무기로 제시합니다.

이를 통해 제안 평가에서 가점을 확보하고 낙찰 확률을 극대화할 수 있습니다.

Phase 3: 온프레미스 엔터프라이즈 솔루션입니다.
금융·공공 기관 대상으로 폐쇄망 격리 어플라이언스로 제품화합니다.

보안 민감 기관의 맞춤형 온프레미스 운영과 라이선스 기반 수익 모델을 실현할 수 있습니다.

사내 효율화에서 출발하여 대외 제안 경쟁력을 거쳐 신사업 수익화로 확대되는 전략입니다.
```
